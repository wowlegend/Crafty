## Testable architecture (the key decision)

Keep Rapier OUT of the store so the **no-permanent-beast invariant is unit-testable without a Rapier world**:
- **The store owns the single-writer form STATE** (`beastFormActive`/`activeBeastForm`) + the enter/exit authority. Every exit path (death / load / manual / timer) routes through `exitBeastForm()`, which clears the state. The store never imports Rapier.
- **Components.jsx owns the imperative collider SWAP.** It subscribes to `activeBeastForm` (a rare transition — isolation-safe, like the M2d `active` SoT) and sets a `pendingFormSwapRef`; the `useFrame` loop consumes that one-shot **immediately before the `collider(0)` read, after all early-returns** (capture/hitstop), does the `setShape` + co-located depenetration + controller-config reset, then clears the ref. The collider restore is verified by the visual gate (plain capsule at base dims = byte-identical capture) + the store-level state tests.

This splits M1 into a **fully-TDD'able core (T1+T2)** and a **gate-verified integration (T3)**.
